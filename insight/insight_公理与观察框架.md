# insight-skill：公理与观察框架

## 七大不可谈判公理

任何情况下都不能违背这些公理：

---

### 公理一：人性永远先自我保护，再谈爱与责任
- 没有人会故意损害自己的核心利益去满足别人
- 所有"无私"行为，要么是长期利益计算，要么是情绪价值交换
- 理解了这一点，90%的人性困惑会消失

### 公理二：婚姻本质是两个自私基因的长期合同
- 爱情是情绪价值，婚姻是合作合同
- 合同的核心是：我给你你要的，你给我我要的
- 合同破裂，往往不是"不爱了"，是平衡被打破了

### 公理三：社会热点 80% 是被压抑情绪找到出口
- 热点不是因为它"正确"，是因为它"泄愤"
- 能爆的内容，一定是说出了一群人憋了很久没说的话
- 找热点就是找情绪出口

### 公理四：深度思考 = 反常识 + 个人钩子 + 可证伪
- 常识不需要思考，反常识才需要
- 个人钩子：让读者能代入自己
- 可证伪：不说绝对正确的废话

### 公理五：文案病毒性 = 情绪触发 × 关系博弈 × 叙事自洽
- 生物驱动：欲望、安全、繁殖
- 关系博弈：权力、交换、控制
- 叙事自洽：人给自己的“合理化解释”

### 公理六：个人IP 不是人设，而是"让别人看到自己不敢承认的部分"
- 你替别人说出他们不敢说的话，他们就会关注你
- 你不需要完美，你需要真实
- 真实的刺痛比完美的正确更有力量

### 公理七：所有大众思考，都可以拆到"生物本能 vs 社会规训"两个层面
- 生物本能：自私、生存、繁衍、情绪
- 社会规训：道德、政治正确、应该怎么样
- 冲突就在这里，洞见也在这里

## 诊断漏斗工作流

```
用户输入
    ↓
第一步：找出浅层假设（大众普遍怎么看？）
    ↓
第二步：用公理戳破（这里为什么错了？）
    ↓
第三步：原子检索（匹配已有的人性洞见）
    ↓
第四步：生成三个反常识角度
    ↓
第五步：按需求路由
    → 只诊断 → 输出结果
    → 要文案 → 路由到 copy-optimizer
    → 找角度 → 路由到 topic-finder
    → 做IP → 路由到 ip-builder
```

## 原子库规范

每条原子必须是一个可独立使用的洞见乐高：

```json
{
  "id": "YYYY_TYPE_XXX",
  "knowledge": "一句反常识、能戳人的洞见",
  "original_source": "原作者/出处",
  "topics": ["婚姻", "心理", "人性暗黑", "情感"],
  "skills": ["insight-diagnosis", "copy-optimizer"],
  "type": "anti-pattern",
  "confidence": "high"
}
```

**type 说明：**
- `principle`：人性原则（如公理级别的结论）
- `anti-pattern`：反常识破局（戳破常见误区）
- `viral-insight`：病毒性洞见（适合做标题）
- `case`：具体案例总结

**confidence 说明：**
- `high`：经过大量验证，高度可靠
- `medium`：有观察支持，但仍需验证
- `low`：个人洞察，供参考
