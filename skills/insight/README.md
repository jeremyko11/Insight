# insight - /insight 热点洞察内容创作工具

OpenClaw skill —— 构建**可复制认知**的生产系统。输入话题，输出结构化诊断 + 完整文案 + 认知模型，一键生成可直接发布的口播内容。

## 核心原理（v1.0）

### 双层认知结构
- **Layer 1：病毒内容** — 流量入口
- **Layer 2：结构认知** — 信任沉淀

### 爆款公式
```
爆款内容 = 情绪触发 × 认知错位 × 可复用结构
```

### 人性拆解三层模型（公理）
**行为 = 生物驱动 × 关系博弈 × 叙事自洽**
- 生物驱动：欲望、安全、繁殖
- 关系博弈：权力、交换、控制
- 叙事自洽：人给自己的"合理化解释"

## 触发方式

```
/insight 你想聊的话题
```

示例：
```
/insight 刷抖音看别人成功很简单，自己做却很难找不到突破口
```

## 完整工作流程（内容进化闭环）

```
输入 → 消解 → 结构化 → 生成 → 反馈 → 更新原子库
```

1. **输入** — 用户/自己观察到的话题（婚姻、情绪、热点事件）
2. **消解** — 逐层诊断，消解伪问题
   - 语言陷阱检测：定义清楚模糊概念
   - 假设错误检测：质疑隐含假设
   - 逻辑错误检测：区分相关性与因果性
   - 事实前提核查：验证基础事实
3. **结构化** — 把洞见归类到固定认知模型
   - 常见模型：控制 vs 依附、交换 vs 幻觉、需求 vs 表达
4. **内容生成** — 按照固定模板输出
5. **数据反馈** — 记录收藏率、评论情绪、转发关键词
6. **反向更新** — 把爆款内容重新原子化，更新原子库

**升级变化**：
- 原：内容生成器
- 新：内容进化系统

## 输出结构（强制模板）

每条内容固定结构：
1. **反直觉开头**（钩子）—— 3秒抓住注意力
2. **人性机制解释**（核心）—— 用认知模型解释
3. **现实场景**（代入）—— 用户能感同身受的例子
4. **反转结论**（记忆点）—— 反常识但符合逻辑
5. **互动问题**（传播点）—— 结尾提问引导评论

## 风格约束

- **人设**：冷静观察者，不是情绪输出者
- **比例**：70% 冷观察 + 20% 反直觉 + 10% 刺激
- **反滥用检查**：避免制造对立、避免过度简化、避免被误读
